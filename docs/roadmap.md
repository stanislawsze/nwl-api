# Roadmap

## Goal

Give contributors and maintainers a shared view of delivery priorities.

## Current phases

### Phase 1 — Foundation

- Dockerized development environment
- PostgreSQL and Redis baseline
- CI quality gates
- code standards and documentation baseline

### Phase 2 — Core platform

- authentication
- roles and permissions
- tenant and server models
- API standards and DTOs

### Phase 3 — FiveM integration

- command model
- bridge authentication
- signed callbacks
- queue-driven command lifecycle

### Phase 4 — Moderation and auditing

- moderation actions
- audit trail
- log querying
- operational support features

### Phase 5 — Realtime and supportability

- Reverb integration
- realtime staff updates
- better observability
- security hardening

## Planning rules

- large features should be broken into multiple issues
- security-sensitive work should not be merged without tests
- architectural changes should be documented through ADRs
